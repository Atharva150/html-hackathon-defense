In these Notes , we will see A Couple Of Important Points & Things About The Internet

  1. Switches

    2. If we want to Connect and Communicate to other device , What is the most Logical Solution to Connect Them ? Connect them with a USB Stick.

    This works for two computers , but what about when we need to talk to 4 devices between them A , B , C & D , we would need cable from A to B , B to C , C to D , A to C and A to D . What about when we have 100 devices? we'd need 4,950 Cables .

    This is not the optimal Solution , and is not Scalable.

    So they Invented a Central Device which is connected by Everyone and everyone communicates to the Switch and sends the message and Switch sends it to the required Device.

  2. Router

    3. LAN(Local Area Network) is a Private Small Network , where devices connect to each other . There are Large Number of Networks operating in their own Private Space. So if I want to send a message to another network?

    We need to connect our network to other networks.For this we have a special dedicated Computer to 'route' the traffic . This device is called a Router

    If we want Networks to connect to each other , we Connect the devices to routers .

    This is a Router

  3. IP Address

    4. IP is the shortform for The Internet Protocol . This is how your Device gets Recognized overt The Internet . Ipv4 is the Classic IP Address everyone uses.

    IPv4 is a 32-bit number

    Example: 172.217.16.142

    A 32-bit bit means it can generate upto 2^32 Possible IP addresses , which is 4.3 Billion .

    To solve The Problem of Limited IP Addresses , IPv6 came in.This uses a 128 bit address .

    This means it can generate upto 340 undecillion


    Types of IP Addresses

    • Private IP Address
    • Public IP Address

    Private IP Address

    A Private IP Address is a IP Address , which is used only within you private network.This is Assigned By Your Router

    Different Devices can have Same Private IP , because they are only used within their network and dont collide with one another

    Public IP Address

    This is The IP Address which is global & unique.

    It is Assigned By your ISP(Internet Service Provider)

    Unlike Private IP Addresses , two devices cannot have same Public Ip address at the same time.

    Because its hard to read , it uses hexadecimal number (0-9) and (a-f)

    Example:2001:0db8:85a3:0000:0000:8a2e:0370:7334

  4. MAC Address

    5. MAC(Media Access Control) Address is a unique , permanent IP Address which is embedded into your Hardware

    It is a 48-bit Address.To make it readable for humans, it is written as 12 Hexadecimal numbers.

    Example:3C:22:FB:A3:B4:C5

  5. Port No.

    6. Port Numbers are numerical identifiers, which are in the range of 0-65535 that specifies a service , application .

    They work with IP Addresses to provide network traffic to a service , to create a Endpoint where users can come .Some of the common are HTTP(port 80) and HTTPS(port 443)

    Categories of Ports:

    1. Well-Known Ports

      2. These are reserved for the most common services on the internet such as :

      • 21: FTP (File Transfer Protocol)
      • 22:SSH(Secure Shell) -
      • 25: SMTP (Simple Mail Transfer Protocol)
      • 53: DNS (Domain Name System)
      • 80:HTTP(HyperText Transfer Protocol)
      • 443:HTTPS(HTTP Secure)
    2. Registered Ports

      3. These ports are for specific applications or services which we can use to register for our software to avoid conflicts with other services.

      Examples:

      • 3000:Unofficial but most commonly used Port for react and node development servers
      • 3306:MySQL Database
      • 5432 - Postgres Database
      • 27017 - Default for MongoDB
      • 8080-Common alternative port for web servers

    3. DNS

      4. DNS(Domain Name System) is a System with human-friendly names which has IP Addresses , without this , we would have to memorize all the IP Addresses of the serevers

      It is Like a Digital Phonebook , Which has IP Addresses of all the systems in the world

      Steps:

      1. Local Cache

        2. Before Checking and going global to Search , It checks into its won Local cache , that whether its present or not.

        1. Browsers keep short-term memory of visited Websites and save the Ip Addresses of the websites
        2. OS(Opearating Systems) also has a cahceh if browser doesn't have it/
        3. router Cache - Routers also have some cache in their own DNS

      2. Recursive Resolver

        3. If the answer is not found , it sends a query to its Recursive DNS Resolver

        This is a DNS run by Your own ISP.

        thsi will recursively search and promises to return what you're looking for.

      3. Hierarchical Search

        4. The Recursive Resolver doesn't have every IP Address , instead it knows how to navigate . It navigates its way from top to bottom like a hierarchical tree.

        1. The Root Servers

          2. It asks to 13 most logical Root servers in the world , where to find the servers that manage the domain.

          The Root Server returns the Address for The TLD servers.

        2. TLD(top-level Domain) Servers

          3. It again asks the same Question to TLD , and TLD returns not teh actual IP Address , but says "I know who does." The Authoritative Name Server

        3. The Authoritative Server

          4. The Resolver asks for teh IP address and this returns teh actual IP Address.

      4. Journey Back

        5. The Authoritative name Server gives The IP address to the Recursive Resolver and it saves it in the local cache , for a period fo time called TTL(Time-to-Live). If anyone asks for the website in the next few hours. It can return it immediately.

        Recursive resolver sends The IP address to the computer's OS which gives it to the Browser.

      5. Final Connection

        6. Now the browser has the actual IP address . It can now open the connection and send a request.
